# 云开发初始化指南

## 问题原因
登录失败是因为云数据库的 `users` 集合尚未创建。云函数不会自动创建集合，需要手动初始化。

## 自动初始化（推荐）

小程序已经配置了自动初始化功能，按以下步骤操作：

### 方法 1：使用 init 页面自动初始化

1. **打开微信开发者工具**
   - 如果已经打开，请关闭当前项目重新打开

2. **清除缓存**
   - 点击菜单 `工具` → `清除缓存` → `清除全部数据`

3. **重新编译**
   - 点击工具栏上的「编译」按钮
   - 会自动打开初始化页面
   - 页面会自动调用云函数创建所有数据库集合
   - 等待 3 秒后自动跳转到首页

### 方法 2：手动运行云函数

如果自动初始化失败，可以手动运行云函数：

1. **打开云开发控制台**
   - 点击工具栏上的「云开发」按钮

2. **找到 init 云函数**
   - 点击左侧菜单「云函数」
   - 找到 `init` 云函数

3. **运行云函数**
   - 右键点击 `init` 云函数
   - 选择「运行」
   - 查看日志确认初始化结果

## 手动创建集合（备选方案）

如果以上方法都失败，可以手动创建集合：

1. **打开云开发控制台**
   - 点击工具栏上的「云开发」按钮

2. **创建数据库集合**
   - 点击左侧菜单「数据库」
   - 点击「+」创建以下集合：
     - `users` - 用户信息
     - `activities` - 活动信息
     - `registrations` - 报名信息
     - `products` - 商品信息
     - `orders` - 订单信息
     - `point_logs` - 积分记录
     - `reviews` - 评价

3. **设置权限**
   - 每个集合的权限设置为「所有用户可读写」

## 初始化完成后测试登录

初始化完成后，返回小程序：

1. 点击「我的」标签页
2. 点击「立即登录」按钮
3. 授权微信用户信息
4. 登录成功！

## 常见问题

### Q: 显示「云函数不存在」
A: 请确认云函数已上传。在微信开发者工具中：
   - 右键点击 `cloudfunctions/init` 文件夹
   - 选择「上传并部署：云端安装依赖」

### Q: 显示「权限不足」
A: 请确认云开发环境 ID 正确（`cloud1-2gyhe7s5efa4155f`），且你有该环境的读写权限。

### Q: 初始化页面一直加载中
A: 可能是云函数未正确上传。请检查：
   - 云函数 `init` 是否存在于云开发控制台
   - 云函数是否已安装依赖
   - 网络连接是否正常

## 技术细节

初始化云函数位于 `cloudfunctions/init/index.js`，主要功能：
- 创建 7 个数据库集合
- 添加示例活动数据
- 添加示例商品数据

小程序启动时会自动调用此云函数（仅第一次），并将结果保存在 `localStorage` 中，避免重复初始化。
