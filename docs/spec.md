# SundayRunningClub 小程序需求分析

## 一、项目概述

为 SundayRunningClub 跑团开发一款微信小程序，实现活动报名、积分管理、周边商城等核心功能。

## 二、用户与角色

### 2.1 角色定义

| 角色 | 权限说明 |
|------|----------|
| 团长 | 最高权限：任命/撤销管理员、解散跑团、设置规则、所有管理员权限 |
| 管理员 | 发布活动、审核报名、签到确认、管理积分、审核新成员、管理周边商品 |
| 团员 | 报名活动、查看积分、兑换周边、查看积分榜 |

### 2.2 用户流程

1. **浏览阶段**：用户可先浏览公开信息（活动列表等）
2. **登录阶段**：微信授权登录
3. **申请阶段**：填写昵称+手机号申请入团
4. **审批阶段**：管理员/团长审批通过后成为正式团员
5. **参与阶段**：正式团员可报名活动、兑换周边等

### 2.3 用户状态

| 状态 | 说明 |
|------|------|
| guest | 游客（未登录） |
| pending | 待审批（已提交申请） |
| member | 正式团员 |
| admin | 管理员 |
| leader | 团长 |

## 三、活动管理

### 3.1 活动类型

- 线下约跑（固定时间地点）
- 支持多种跑步类型（晨跑、夜跑、长距离跑等）

### 3.2 活动属性

| 属性 | 类型 | 说明 |
|------|------|------|
| title | string | 活动标题 |
| description | string | 活动描述 |
| location | string | 集合地点 |
| start_time | datetime | 开始时间 |
| end_time | datetime | 结束时间 |
| quota | number | 名额限制 |
| points | number | 参与积分 |
| status | string | 活动状态 |

### 3.3 活动状态

| 状态 | 说明 |
|------|------|
| draft | 草稿 |
| published | 已发布 |
| ongoing | 进行中 |
| ended | 已结束 |
| cancelled | 已取消 |

### 3.4 活动流程

1. 管理员创建活动（草稿状态）
2. 管理员发布活动（已发布状态）
3. 团员报名参加
4. 活动开始（进行中状态）
5. 管理员现场签到
6. 活动结束，积分自动发放

### 3.5 报名规则

- 团员可自由报名（有名额限制）
- 报名后可取消（活动开始前）
- 签到后获得对应积分

## 四、积分系统

### 4.1 积分规则

| 功能 | 说明 |
|------|------|
| 积分获取 | 参加活动签到后获得（按活动类型设置不同积分） |
| 积分有效期 | 永久有效，持续累积 |
| 积分用途 | 兑换周边商品 |

### 4.2 积分类型

| 类型 | 说明 |
|------|------|
| activity | 活动签到获得 |
| exchange | 兑换商品消耗 |
| admin | 管理员手动调整 |

### 4.3 积分榜

- 展示所有团员的积分排名
- 支持按时间范围筛选

## 五、周边商城

### 5.1 商品属性

| 属性 | 类型 | 说明 |
|------|------|------|
| name | string | 商品名称 |
| description | string | 商品描述 |
| image | string | 商品图片 |
| points_price | number | 积分价格 |
| cash_price | number | 现金价格（分） |
| stock | number | 库存数量 |
| status | string | 商品状态 |

### 5.2 购买方式

1. **积分兑换**：使用积分直接兑换
2. **微信支付**：现金购买

### 5.3 订单流程

1. 用户选择商品
2. 填写收货地址
3. 确认支付方式
4. 完成支付
5. 等待发货
6. 确认收货

### 5.4 订单状态

| 状态 | 说明 |
|------|------|
| pending | 待支付 |
| paid | 已支付 |
| shipped | 已发货 |
| completed | 已完成 |
| cancelled | 已取消 |

## 六、技术要求

### 6.1 前端要求

- 微信小程序原生开发
- 适配主流手机屏幕
- 流畅的用户体验

### 6.2 后端要求

- 微信云开发
- 云函数实现业务逻辑
- 云数据库存储数据
- 云存储存储图片

### 6.3 安全要求

- 敏感操作需要权限验证
- 用户数据加密存储
- 支付安全验证

## 七、非功能性需求

### 7.1 性能要求

- 页面加载时间 < 2秒
- 接口响应时间 < 1秒

### 7.2 可用性要求

- 界面简洁直观
- 操作流程清晰
- 错误提示友好

### 7.3 兼容性要求

- 支持 iOS 和 Android
- 适配微信最新版本
