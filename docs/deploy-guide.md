# 云函数部署指南

## ⚠️ 重要提示

由于微信开发者工具 CLI 工具的网络兼容性问题，**建议使用微信开发者工具 UI 手动上传**云函数。这是最稳定可靠的方式。

## 📖 手动部署步骤

### 步骤 1：打开微信开发者工具

1. 启动 **微信开发者工具**
2. 加载本项目（`/Users/xu/Documents/SRC_wechat`）

### 步骤 2：选择云环境

1. 在顶部菜单栏，点击 **"工具"** → **"云开发"**
2. 在云开发控制台右上角，确认当前环境是 `cloud1-2gyhe7s5efa4155f`
3. 如果不是，点击下拉菜单选择正确的环境

### 步骤 3：上传 test-user 云函数

1. 在左侧文件树中，展开 `cloudfunctions/` 目录
2. 找到 `test-user` 文件夹
3. **右键点击** `test-user` 文件夹
4. 选择 **"上传并部署：云端安装依赖"**
5. 等待上传完成（状态栏会显示"上传成功"）

### 步骤 4：上传 test-data 云函数

1. 在左侧文件树中，找到 `test-data` 文件夹
2. **右键点击** `test-data` 文件夹
3. 选择 **"上传并部署：云端安装依赖"**
4. 等待上传完成

### 步骤 5：验证部署

1. 在微信开发者工具顶部，点击 **"云开发"** 按钮
2. 进入云开发控制台
3. 点击左侧 **"云函数"** 菜单
4. 查看函数列表，确认 `test-user` 和 `test-data` 都在列表中
5. 状态应该是 **"部署成功"**

## 🚀 使用测试工具

部署完成后：

1. 在小程序中，进入 **"我的"** 页面
2. 点击 **"🛠️ 测试工具面板"** 按钮
3. 点击 **"创建测试用户"**
4. 点击 **"初始化测试数据"**
5. 点击任意测试用户卡片切换身份

## ❓ 常见问题

### Q: 右键菜单没有"上传并部署：云端安装依赖"选项

**A:** 确保：
- 已在项目配置中设置了云环境 ID
- 已登录微信开发者账号
- 云函数目录包含 `package.json` 和 `index.js`

### Q: 上传后状态显示"部署失败"

**A:** 点击云函数查看错误日志，常见问题：
- 网络问题：重试上传
- 依赖问题：选择"云端安装依赖"而非"上传"
- 代码错误：检查 `index.js` 语法

### Q: 小程序中调用云函数返回"函数不存在"

**A:** 等待 1-2 分钟让云函数生效，或：
1. 在云开发控制台找到该函数
2. 点击"更多" → "重新部署"

## 🔧 云函数说明

| 云函数 | 用途 | 主要功能 |
|--------|------|----------|
| test-user | 测试用户管理 | 创建、切换、重置测试用户 |
| test-data | 测试数据管理 | 创建、清理、重置测试数据 |
| debug | 数据库调试 | 诊断用户状态、修复角色 |
| init | 初始化 | 创建数据库集合和示例数据 |
| user | 用户管理 | 登录、审批、角色设置 |
| activity | 活动管理 | 创建、报名、签到 |
| admin | 管理后台 | 成员、活动、商品、订单管理 |
